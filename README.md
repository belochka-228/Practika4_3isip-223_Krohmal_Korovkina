## Архитектура приложения
### Главное окно (MainWindow)
- В верхней части расположены три кнопки для переключения между страницами.
- В центре находится элемент `Frame`, который динамически загружает выбранную страницу.
- При запуске открывается первая страница.
- При попытке закрыть окно появляется диалог подтверждения выхода.

### Страницы (папка Pages)
Все страницы унаследованы от `Page` и хранятся в отдельной папке `Pages` для удобства навигации по проекту.

**Общие элементы каждой страницы:**
- Изображение формулы (элемент `Image` с путём к файлу из папки `Images`).
- Поля ввода (`TextBox`) для соответствующих переменных.
- Поле вывода результата (`TextBox` только для чтения).
- Кнопки «Вычислить» и «Очистить» (с всплывающими подсказками).
- Проверка ввода: если поле пустое или содержит нечисловое значение, выводится сообщение об ошибке.

**Особенности страниц:**
- **Страница 1:**  
  Три поля ввода (x, y, z). Перед расчётом проверяется, что sin(z) не равен нулю, и подкоренное выражение неотрицательно.
- **Страница 2:**  
  Поля x и q. Добавлены три радиокнопки для выбора f(x): sh(x), x², eˣ. В зависимости от |x·q| выбирается одна из трёх формул.
- **Страница 3:**  
  Поля x₀, xₖ, dx, b.  
  – После нажатия «Вычислить» строится таблица значений в многострочном `TextBox`.  
  – Для отображения графика используется `WindowsFormsHost`, внутри которого размещён элемент `Chart` из Windows Forms.  
  – Проверяется, что выражение под корнем (x + b – 0.0084) ≥ 0 и шаг dx ≠ 0.  
  – График автоматически масштабируется по осям и окрашивается в синий цвет.

### Навигация
Переключение страниц реализовано через обработчики кнопок в главном окне:
```csharp
MainFrame.Navigate(new Pages.Page1());
```
## Проверка ввода и всплывающие подсказки
- Для каждого TextBox задано свойство ToolTip с краткой подсказкой.
- При нажатии «Вычислить» данные проверяются с помощью double.TryParse(). При ошибке выводится MessageBox.
- На третьей странице дополнительно контролируется область определения корня.

### В ходе работы был выполнен рефакторинг трёх страниц WPF-приложения. Из каждого обработчика btnCalc_Click была вынесена математическая логика в отдельный статический метод.

### Причина успешного выполнения тестов
- Правильный рефакторинг – математическая логика отделена от UI, что позволило тестировать её изолированно.
- Корректные ожидаемые значения – тесты используют данные, полученные вручную при работе приложения, что гарантирует соответствие ожидаемого результата реальным вычислениям.
- Использование статических методов – методы для тестирования объявлены как public static, что даёт возможность вызывать их из тестового проекта после добавления ссылки на основной проект.
